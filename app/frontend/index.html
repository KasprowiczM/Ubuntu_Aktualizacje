1. Install

        
Three install paths, pick whichever fits:

        
# A — fresh machine (one-liner)
git clone https://github.com/KasprowiczM/Ubuntu_Aktualizacje ~/Dev_Env/Ubuntu_Aktualizacje
cd ~/Dev_Env/Ubuntu_Aktualizacje
bash scripts/fresh-machine.sh

# B — Debian package (.deb)
bash packaging/build-deb.sh
sudo dpkg -i dist/ascendo_*_all.deb
ascendo --help

# C — manual setup (advanced)
bash setup.sh
bash app/install.sh                       # creates app/.venv with FastAPI
bash systemd/user/install-dashboard.sh    # autostart dashboard at login


        
2. First run

        
The wizard at http://127.0.0.1:8765 on first dashboard
        visit asks for: language, default profile (quick/safe/full), weekly
        schedule, and snapshot-before-apply preference. CLI:

        
./update-all.sh                       # full profile, NVIDIA held
./update-all.sh --profile quick       # read-only sweep (~15s)
./update-all.sh --profile safe        # full pipeline, no driver mutations
./update-all.sh --nvidia              # apply NVIDIA driver upgrade too
./update-all.sh --snapshot            # timeshift/etckeeper before apply
./update-all.sh --budget 30m          # stop after 30 min, resume next time


        
3. CLI cheat-sheet

        
ascendo                              # equivalent of ./update-all.sh
ascendo fresh                        # bring-up on a new machine
ascendo dashboard                    # start dashboard ad-hoc
ascendo schedule --calendar "Sun *-*-* 03:00:00" --profile safe
ascendo snapshot create "before risky run"
ascendo snapshot list
ascendo apps detect                  # tracked vs detected vs missing report
ascendo apps add <pkg> --category snap
ascendo apps install-missing
ascendo settings export backup.tar.gz
ascendo settings import backup.tar.gz
ascendo health --json
ascendo exclusions list / add / remove cat:pkg


        
4. Scripts reference

        
Every category implements 5 phases (check, plan,
        apply, verify, cleanup) emitting
        a JSON sidecar to logs/runs/<id>/<cat>/<phase>.json (schema v1).

        

          
ScriptWhat it does

          
            
update-all.shMaster orchestrator. Reads --profile, locks /run/user/<uid>/ubuntu-aktualizacje.lock, prompts sudo ONCE, builds askpass helper, runs all categories × all phases, writes run.json, emits CHECK-ONLY banner if no apply phase.

            
scripts/apt/*.shAPT: refresh repos (fail-closed), apt-get upgrade with per-package streaming progress, NVIDIA hold by default, exclusions via apt-mark hold for the duration of the phase.

            
scripts/snap/*.shSnap: snap refresh with --ignore-running fallback for browsers, removes disabled revisions in cleanup.

            
scripts/brew/*.shHomebrew: drops to SUDO_USER (brew refuses root), proactive Cellar chown to fix pipx pyc cleanup warnings.

            
scripts/npm/*.shnpm globals: forces @latest for AI CLIs (Claude Code, Codex, Gemini CLI), npm audit in verify.

            
scripts/pip/*.shpip user packages + pipx upgrade-all; respects PEP-668 by routing system-managed Python through pipx.

            
scripts/flatpak/*.shflatpak update + cleanup of unused runtimes.

            
scripts/drivers/*.shNVIDIA driver (held unless --nvidia), fwupd metadata refresh + manual update advisory.

            
scripts/inventory/*.shRegenerates APPS.md (gitignored) — installed packages by category.

            
scripts/snapshot/create.shPre-apply snapshot: timeshift (300s timeout) → etckeeper fallback. Never blocks the run.

            
scripts/scheduler/install.shInstalls systemd timer ubuntu-aktualizacje@%i.timer with ExecStartPre=should-run.sh.

            
scripts/scheduler/should-run.shPre-flight gate: defers if on battery (UA_REQUIRE_AC=1), outside maintenance window (UA_MAINTENANCE_WINDOW=02:00-05:00), or apt/dpkg busy.

            
scripts/health-check.shPost-run audit: systemctl --failed, dmesg errors, disk pressure, reboot pending. Emits score 0–100 to health.json.

            
scripts/apps/{detect,add,remove,install-missing,list}.shApp registration: detect installed-but-untracked, add to config/<cat>-packages.list, remove, bulk-install missing.

            
dev-sync/dev_sync_export.pyPush private overlay (.env.local, agent settings, keys) via rclone to your cloud provider. Copy semantics, never sync-with-deletes.

            
scripts/maintenance/prune-logs.shLog retention: keep last N runs OR last N days (whichever larger).

            
scripts/notify.shNotification routing: ntfy, Slack webhook, email, Telegram, plus desktop notify-send.

6. Dashboard tour

        
Sidebar items, top-down: Overview (charts + quick actions),
        Categories (per-category 5-phase buttons + add-package widget),
        Run Center (start a run, see live progress bar), History
        (past runs + ETA averages), Logs (per-phase JSON + plain log viewer),
        Sync (git push/pull + dev-sync export + cloud provider config),
        Apps (tracked vs detected, exclusion checkboxes), Suggestions
        (heuristic + AI advice as diffs), Hosts (multi-host preflight),
        Settings (defaults, theme, scheduler, backup), Help (this page),
        About (version + release notes).


        
7. Scheduler & maintenance windows

        
Settings → Scheduler installs a per-user systemd timer. Set env in
        ~/.config/environment.d/ascendo.conf to make it skip on
        battery / outside hours:

        
UA_REQUIRE_AC=1
UA_MAINTENANCE_WINDOW=02:00-05:00
UA_RESPECT_FOCUS=1


        
8. Snapshots & rollback

        
With --snapshot (or the toggle in Settings), Ascendo
        creates a timeshift snapshot before apply. timeshift hangs without a
        TTY are bounded by a 300s timeout — the run never stalls. To roll back:
        POST /snapshots/restore with {"id":"<snap-id>","confirm":"RESTORE"}
        or ascendo snapshot list & scripts/snapshot/restore.sh <id>.


        
9. Dev-sync (cloud overlay)

        
Workflow separate from updates. GitHub holds tracked files. Cloud
        provider (Proton, Google Drive, Dropbox, OneDrive, WebDAV/Nextcloud,
        S3, local) holds a small private overlay (~8 files: tokens, agent settings).
        Sync screen lets you pick the provider; first-time auth happens via
        bash dev-sync/provider_setup.sh in a real terminal because
        rclone needs OAuth callback. Then bash dev-sync-export.sh
        --dry-run & the real export.


        
10. AI Suggestions

        
Heuristic engine runs free, deterministic rules over inventory + run
        history. Optional LLM step *enriches* rationales — never edits files
        autonomously. Supports Anthropic (Claude), OpenAI, Google Gemini,
        Ollama (local), LM Studio (local), or any OpenAI-compatible endpoint.
        Local providers need no API key; just set the Base URL (e.g. Ollama:
        http://127.0.0.1:11434, LM Studio: http://127.0.0.1:1234/v1).
        All apply paths show a unified diff with manual confirm.


        
11. Troubleshooting

        
    
          
  • "a run is already in progress" 409 — POST /runs/active/stop or remove stale lock $XDG_RUNTIME_DIR/ubuntu-aktualizacje.lock after verifying no process holds it.
    • 
          
  • NVIDIA DKMS sypie po upgrade — bash scripts/rebuild-dkms.sh && reboot.
    • 
          
  • apt-get update failuje — najczęściej duplikat *.list + *.sources; lib/repos.sh łapie i naprawia automatycznie.
    • 
          
  • Snap firefox blocked — close Firefox, then --only snap --phase apply; or accept the --ignore-running fallback (forces refresh while running).
    • 
          
  • Frontend pusta strona — Ctrl+Shift+R, sprawdź /health w curl, sprawdź journalctl --user -u ubuntu-aktualizacje-dashboard.
